CDAT to IDX On-Demand Data Conversion Demo Page

These are a selection of datasets available on pcmdi11.llnl.gov.
For each there is a cdat .xml file generated using cdscan.
Selecting a dataset for the first time will generate the associated IDX volume headers, register these new volumes with the ViSUS server, and return a configuration file specifying the url of these new volumes.
Successive selections will simply return the config file with the volume urls.
There is one IDX volume per domain of the climate dataset.
The actual data is converted to the hierarchical IDX format only when it is requested by a client.
Once a dataset is loaded by the client, the current variable and timestep are automatically converted by the server and sent on.
The conversion is cached so successive requests can be immediately satisfied, amortizing the cost of the conversion.
The conversion time depends primarily on the domain size.
Even though some of these datasets are hundreds of gigabytes in size, their respective domains are small enough that conversion rarely requires more than a fraction of a second.
However, for very large domains, converting data to a hierarchical format conveys the advantage of streaming multiresolution access.
